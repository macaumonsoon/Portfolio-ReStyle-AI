# Portfolio ReStyle AI — 项目背景与产品说明

## 定位与使用者

面向 **视觉设计师**（及个人工作室）的 **AI 网页端设计辅助工具**：在浏览器中 **预览** 并 **辅助重排** 已有作品集，接近「作品集自动排版软件」的体验，但强调 **AI 是协作者**——设计师保留对风格、版式方向与每一页版本的控制权。

**构建方式**：使用 **Cursor** 等工具迭代开发；功能实现须符合本文档流程与原则。

## 核心价值（一句话）

用户 **自主上传** 原有作品集（**PDF 或 SVG，由用户选择**），系统基于 **网格系统** 对内容分区并 **重新排列**；用户补充 **生成想法** 与 **选项**（画布尺寸、风格、字体风格等）；**AI 助手** 按 **作品集逻辑顺序** 或 **生活 / 工作叙事顺序** 逐页生成新版式；**每一新页自动生成 3 个版本**，用户 **每页重新选择其一** 后再进入下一页；全程 **网页端预览**，并可导出可继续编辑的交付物（以 SVG / Figma 友好结构为目标）。

## 参考案例

真实排版与层级参考：**Bold Studio《Capabilities Deck 2025》**（Figma 导出 SVG 类结构）。

## 用户体验流程（须严格遵守）

1. **上传**  
   - 用户 **自主选择** 上传 **PDF** 或 **SVG**（不设强制优先级；实现上可因格式差异分阶段支持，但产品描述上二者对等）。  
   - 系统将文件解析为可操作的页面与元素（PDF 需经 pdf.js 或服务端转为逐页可布局表示，目标与 SVG 管线对齐）。

2. **输入想法与选项**  
   - **文字/想法**：描述希望的整体气质、重点信息、叙事侧重等（可与下方选项一起作为 AI 条件）。  
   - **画布尺寸**：如 A4、Letter、方屏、自定义等。  
   - **风格**：关键词或预设（如 Bold、Minimal、叙事感、复古、优雅等）。  
   - **字体风格**：方向性选择（无衬线、人文、衬线编辑感、等宽等）。  
   - 可选：**色系**、**网格密度** 等（若产品后续扩展，须仍服从「内容不丢、可预览、可导出」原则）。

3. **AI 处理逻辑**  
   - 使用 **网格系统（Grid）** 将原始作品集 **分区**（区块、标题、图、正文层级等）。  
   - 在 **保留原文案与图片资源** 的前提下，做 **重排、间距与层级** 调整，而非替换内容。  
   - 页面顺序与单页结构服从：用户选择的 **作品集逻辑顺序** 或 **生活 / 工作逻辑顺序**（与上一步「想法」一致）。  
   - **每生成新的一页**，输出 **3 个不同版本**；用户 **必须选定其中一个** 后，才生成 **下一页**（每页都重新三选一，保证逐步对齐设计师偏好）。

4. **网页预览**  
   - 所有关键步骤可在 **网页内实时预览**（版式与样式变化）。  
   - 建议支持 **Before / After** 或分模式对比（如颜色 vs 版式），便于设计师判断。

5. **输出**  
   - 导出 **干净、可编辑的 SVG**（文字为 `<text>`、图片为 `<image>`，层级清晰），并保留 **Figma 友好** 路径（如结构化 JSON / 插件）。  
   - **不得以 WebGL 截屏作为正式矢量交付物**（见下文 Three.js 说明）。

## AI / Cursor 生成代码时的行为约束

- **每新生成一页前**，须参考 **用户已在上一页选定的版本**，保持 **全书风格与栅格节奏** 一致。  
- **100% 保留** 原始文字与图片 **内容**；仅调整 **位置、尺寸、颜色、排版与网格**。  
- 前端布局与预览中的页面结构：**现代栅格（CSS Grid + 弹性间距）**；SVG 管线以 **原生 SVG DOM** 与 **svg.js / @svgdotjs/svg.js** 为主。  
- 状态上须显式建模：**当前页索引、每页 3 版本、每页用户选择结果、再进入下一页**。

## 技术栈要点

- **前端**：Next.js 14+ App Router、Tailwind CSS、shadcn/ui（或同级组件）。  
- **SVG**：解析、变换与导出以 **SVG 为权威数据源（SSOT）**。  
- **PDF**：pdf.js 或后端转多页 SVG / 中间结构，与网格管线对齐。  
- **AI**：多模态/结构化输出须能对接「网格块 + 页面顺序 + 用户选项」。  
- **状态**：多页、每页三版本、选择与解锁下一页，须可测试、可扩展。

## 关于 Three.js 与本文档的关系（审阅说明）

- **`background.md`（本文）**：定义 **产品流程、设计师体验、AI 与网格职责、SVG/PDF 主线与导出原则**。  
- **本文不包含「如何用 Three.js 搭建整个产品」的逐步教程**；版面权威与可编辑导出 **不建立在 Three.js 上**。  
- **Three.js 的用途边界、与 SVG 导出对齐、Next.js SSR、性能与资源释放** 等 **专门说明** 见同目录 **`technical-three.md`**。  
- 若使用 Three.js，仅作为 **可选预览 / 动效层**（例如纹理烘焙预览、展示型动效），且须满足：`technical-three.md` 与本文 **第 5 节导出原则**。

## 设计原则

- 界面 **极简、专业**，符合设计师日常工具审美。  
- 步骤清晰：**上传 → 想法与选项 → 逐页预览 → 三选一 → 下一页 → 导出**。  
- **AI 可解释、可打断**：用户每页都能用选择纠正方向。  
- 输出物应达到 **可直接向客户展示** 的完成度（在设计师微调前也体面可用）。
