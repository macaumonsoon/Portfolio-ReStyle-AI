Portfolio ReStyle AI

Design-focused presentation for a visual communication design workflow tool.

視覺傳達設計流程工具的設計向簡報（課程展示用）。

Why this app

為什麼做這個工具

Portfolios are not only visuals—they are narratives of process and identity. Designers spend heavy time on repetitive layout work (grids, scaling, reordering) instead of storytelling. This prototype explores human–AI collaboration: upload existing work, set intent (palette, style, narrative order), compare variants, and export—keeping the designer in control.

作品集不只是畫面，更是敘事與專業身份。設計師常把大量時間花在網格、縮放、重排等重複勞動上。本原型探討人機協作：上傳既有稿件、設定意圖（色系、風格、敘事順序）、比對多版本再匯出，決策權仍在設計師手中。

Use in visual communication design

在視覺傳達設計流程中的用途

  • Restructure multi-page PDF/SVG decks for different audiences (interview, review, print).
  • Explore palette and style directions quickly with live preview before deep production.
  • Iterate on grid-based compositions (remix + optional tile nudge) as a layout sketching aid.
  • Document decisions via export (SVG for Figma/editor, PDF for sharing).
  • 為不同場景（面試、評圖、印刷）重組多頁 PDF／SVG 稿。
  • 在深入製作前，用即時預覽快速試驗色系與風格方向。
  • 以網格重排與區塊微調作為版面草圖手段，反覆嘗試構圖。
  • 透過匯出（向量 SVG／合併 PDF）固化設計決策與交付物。

Design thinking

設計思維過程

We framed the problem as re-interpretation, not full automation: the system proposes variations; the designer selects. The flow mirrors real studio steps—upload → intent (style/canvas) → review per page → export—so cognitive load stays low and trust stays high.

我們將問題定義為再詮釋而非全自動：系統提出變體，設計師做選擇。流程對齊真實工作室步驟—上傳 → 意圖（風格／畫布）→ 逐頁審閱 → 匯出—以降低認知負荷並維持信任感。

Key UI/UX decisions

主要介面／體驗決策

  • Bilingual toggle in the header—persistent via localStorage for global audiences and coursework requirements.
  • Stepper wizard reduces overwhelm; advanced options (grid, narrative) stay on the style step.
  • Dock preview stays visible while editing options; export uses full-strength styling so previews stay readable.
  • Three variants per page supports comparison without infinite choice paralysis.
  • Before/After + pan helps judge color vs. layout separately.
  • Grid preset icons (mini diagrams) make asymmetric layouts (1×2, 2×1) scannable.
  • 頂部雙語切換並寫入 localStorage，兼顧課程要求與跨語境使用者。
  • 步驟式精靈降低資訊量；網格、敘事等進階選項集中在風格步。
  • 右側 dock 預覽與選項同步；匯出仍保留完整濾鏡強度，預覽則偏清晰可讀。
  • 每頁三版本利於比對，又避免選項過多造成決策疲勞。
  • Before/After 與平移可分開檢視色彩與版式差異。
  • 網格預設圖示（小示意圖）讓 1×2、2×1 等非對稱網格一眼可辨。

Development note (brief)

開發說明（簡述）

Built with Next.js, Tailwind, Zustand; optional Gemini APIs for copy and ordering assistance. Technical choices support reliable SVG/PDF handling in the browser; emphasis in grading should remain on design rationale above implementation detail.

以 Next.js、Tailwind、Zustand 實作；可選 Gemini API 輔助文案與排序。技術選型服務瀏覽器內 SVG／PDF 處理；評分重點仍建議放在設計論述而非純技術細節。
