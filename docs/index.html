Problem

問題定義

Portfolios are narrative and craft, but deck restructuring (order, grid, tone) is repetitive and steals time from concept work.

作品集是敘事與工藝，但頁序、網格、色調等重整工作高度重複，壓縮了概念發想時間。

Concept / Why this app

核心概念 / 為何做這個工具

Re-interpret existing SVG/PDF with clear intent. System generates variants; designer chooses, then exports SVG/PDF deliverables.

以明確意圖重詮釋既有 SVG/PDF：系統生成變體、設計師做選擇，最後匯出 SVG/PDF 交付物。

UX 流程（對齊簡報）

流程 2選項：風格、色系、畫布、敘事
流程 3頁面：比較三種版本
流程 4匯出：合併 PDF + 每頁 SVG

Role in VCD Process

在視傳流程中的角色

  • Direction exploration before deep production.
  • Grid remix and tile nudge as layout sketching.
  • Audience-specific packaging via page reorder and style consistency.
  • 在深入製作前先做方向探索。
  • 用網格重排與 tile 微調作為版面草圖工具。
  • 透過頁序重組與風格一致化，形成不同受眾版本。

Design Thinking Process

設計思維流程

  • Empathize: fear of losing control to black-box AI.
  • Define: automate packaging and variation, not authorship.
  • Ideate: wizard + dock preview + three variants.
  • Prototype/Test: bilingual flow, preview clarity, export reliability.
  • 同理：設計師擔心被黑箱 AI 取代控制權。
  • 定義：自動化包裝與變化，而非取代作者性。
  • 發想：精靈流程 + 側欄預覽 + 三版本比較。
  • 原型/測試：雙語流程、預覽清晰度、匯出穩定性。

Key UI/UX Decisions

關鍵 UI/UX 決策

  • Stepper wizard reduces cognitive load.
  • Persistent preview dock keeps feedback immediate.
  • Bilingual toggle in header for mixed-locale users.
  • Only 3 variants to avoid choice overload.
  • Before/After separates look and composition judgment.
  • 步驟精靈降低資訊負擔。
  • 持續可見的 dock 預覽強化即時反饋。
  • 頁首雙語切換支援跨語境使用者。
  • 每頁固定三版本，避免選擇過載。
  • Before/After 分離視覺語氣與構圖判斷。

Reflection & Next Step

反思與下一步

Next, we plan finer typography controls, broader accessibility checks, and stronger layout intelligence while keeping API keys local-only.

下一步將加強字體微調能力、完整可及性檢查與版面智能，同時維持 API key 僅本機使用。
