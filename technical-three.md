# Portfolio ReStyle AI — Three.js 預覽與互動層規範

本文檔說明 **Three.js 在本產品中的邊界、與 SVG 主線的關係、匯出對齊方式，以及 Next.js / 效能約束**。實作時須與根目錄 `background.md` 的產品流程一併遵守。

---

## 1. 文件目的與範圍

- **目的**：允許以 Three.js 作為 **預覽與互動** 的主要呈現手段（例如翻頁、景深、環境光、微動效），同時 **不破壞**「可編輯 SVG + Figma 相容匯出」這條主交付鏈。
- **範圍**：僅規範前端預覽層；**AI 版式邏輯、SVG 解析、最終匯出格式**仍以 `background.md` 為準。

---

## 2. Three.js 在產品中的角色（必須明確其一或組合）

實作前須在產品/技術上鎖定用途，避免 Three 與 SVG 雙主線打架：

| 模式 | 說明 | 典型用途 |
|------|------|----------|
| **僅預覽** | 畫面以 WebGL 呈現，但 **幾何與樣式來自對 SVG 的同步或烘焙**；使用者操作的仍是「同一套頁面資料」。 | 大畫布流暢縮放、多頁畫廊、對比滑桿疊加前的 GPU 合成。 |
| **僅動效** | 內容仍以 **DOM / inline SVG** 為主，Three 只負責背景、粒子、轉場等非版面元素。 | 品牌感背景、頁面切換過場；**不承載**可編輯文字方塊邏輯。 |
| **混合（預覽 + 動效）** | 預覽主舞台用 WebGL，動效與預覽共用同一場景圖；**匯出仍只認 SVG 管線**。 | 高階展示模式；需嚴格遵守第 4 節對齊規則。 |

**原則**：Three.js **不是**版面與內容的權威來源；權威來源始終是 **結構化 SVG（或從 PDF 轉出的等價 SVG 頁）**。

---

## 3. 與 SVG 的關係：單一事實來源（SSOT）

- **資料**：每一頁的文案、圖片引用、柵格區塊、色系 token，應存在於可序列化的 **SVG / 中間層 JSON**（與 `background.md` 一致），而非只存在於 Three 的 `Mesh`/`Material` 狀態裡。
- **預覽**：Three 層可透過下列方式之一與 SVG 對齊（擇一並寫進實作文檔）：
  - **紋理烘焙**：將當前頁 SVG（或向量柵格化後的向量渲染結果）作為 `CanvasTexture` / 高 DPI 點陣貼圖更新（注意文字銳利度與色彩管理）。
  - **幾何同步**：將解析後的矩形/路徑等映射為平面網格（維護成本高，僅在確有需要時採用）。
- **禁止**：在 Three 場景內直接「手改文案」作為唯一狀態，卻未回寫 SVG——會導致 **匯出與預覽不一致**。

---

## 4. 與 SVG 匯出對齊（硬性規則）

1. **匯出檔**：使用者點擊「匯出」時，**只從 SVG SSOT**（及 Figma 相容附帶結構）生成檔案；**不得**從 WebGL framebuffer 截圖當作正式 SVG 交付物。
2. **Before / After**：若對比預覽在 Three 中完成，**「After」狀態**必須對應已套用版式/色系變更後的 **SVG 字串或 DOM**，再決定是同步到紋理還是並排 2D 預覽。
3. **「僅顏色 / 僅版式」切換**：邏輯上應作用於 **SVG 層的 token 與佈局屬性**；Three 層僅負責顯示結果一致，避免兩套配色邏輯。
4. **圖層與 Figma**：最終 `<text>`、`<image>`、群組層級仍以 SVG 管線為準；Three 場景樹 **不替代** Figma 可編輯結構。

---

## 5. Next.js 14+ App Router 與 SSR 約束

- **僅客戶端**：Three.js、`requestAnimationFrame`、WebGL context 必須在 **Client Component** 內初始化；預設 **`dynamic(..., { ssr: false })`** 或等同策略，避免伺服端渲染時建立 WebGL。
- **程式分割**：使用 `next/dynamic` 懶加載預覽模組，避免首屏 bundle 過大。
- **`useEffect` 守衛**：在 `typeof window !== 'undefined'` 且容器已掛載、尺寸已知後再 `new THREE.WebGLRenderer`；卸載時 `dispose()` 幾何、材質、紋理與 renderer。
- **React Strict Mode**：開發環境可能雙掛載，需保證 **清理邏輯冪等**，避免重複 canvas 或記憶體洩漏。
- **路由**：離開預覽頁時暫停渲染迴圈（`cancelAnimationFrame`），降低背景分頁耗電與發熱。

---

## 6. 效能與體驗

- **紋理更新**：全頁 SVG 每幀重繪為紋理成本極高；應在 **資料變更** 或 **對比滑桿拖動節流** 時更新，而非無條件每幀。
- **解析度**：依容器 DPR 縮放，提供 **上限**（例如 max 2x 或依裝置降級），避免 4K 紋理爆記憶體。
- **行動端**：預設可降為 **2D SVG 預覽** 或降低 Three 效果；提供開關或自動偵測 `webgl` 失敗時回退。
- **多版本（每頁 3 選 1）**：每個版本的 **預覽** 可共享同一解析管線，避免三份完整 WebGL 場景長駐；切換版本時以 SSOT 差分更新為主。
